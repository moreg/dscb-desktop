# 技能 v3.2 Markdown 格式规范

> 本文档列出 app 解析器对 `.md` / `.html` 文件的**硬性格式要求**——哪些写法必须遵守（否则丢数据），哪些容错。
> 解析器源码：[src/main/data/skill-format/](../src/main/data/skill-format/)
> 参考样本：`O:\book\测试写作\`（v3.2 完整 13 文件）

---

## 一、设计原则

1. **遵循标准 GFM（GitHub Flavored Markdown）** —— `#`/`##`/`###` 标题、`|` 管道表格、`-` 列表。
2. **技能 v3.2 产出的文件零风险** —— 解析器照着样本写，25 个测试覆盖。
3. **容错优先于严格** —— 缺字段返回 `undefined`，不崩；未知节/列原样保留。
4. **写回保真** —— 外科手术式替换，版本头 / H1 / 未识别节 / 代码块 / 表头不动。

---

## 二、通用硬性规则（所有 .md 文件适用）

### 2.1 标题必须带空格

| 正确 | 错误（不认） |
|------|-------------|
| `# 标题` | `#标题` |
| `## 标题` | `##标题` |
| `### 标题` | `###标题` |

**正则**：`^# (.+?)\s*$` / `^## (.+?)\s*$` / `^### (.+?)\s*$`（`#` 后必须有 1 个空格）

**影响**：标题层级靠空格后的 `#` 数量区分。`##标题` 不会被识别为 H2，其内容不会归入任何节 → **整节丢失**。

### 2.2 字段行必须是 `- **字段**：值` 形态

**正则**：`^\s*-\s+\*\*(.+?)\*\*\s*[：:]\s*(.*)$`

| 正确 | 错误（字段不被解析） |
|------|---------------------|
| `- **姓名**：苏铭` | `姓名：苏铭`（无 `- **`） |
| `- **身份**: 大学生`（英文冒号也行） | `- 姓名 - 苏铭`（用破折号分隔） |
| `-   **身份**：大学生`（多空格也行） | `**姓名**：苏铭`（无 `- ` 列表标记） |

**冒号**：中文 `：`（U+FF1A）或英文 `:` 都认。

### 2.3 多行子列表必须缩进 ≥2 个空格

**正则**：`^\s{2,}-\s+(.+)$`（行首 ≥2 个空白字符 + `- `）

| 正确 | 错误（子项被丢） |
|------|-----------------|
| ``` - **行为习惯**：<br>&nbsp;&nbsp;- 每天...<br>&nbsp;&nbsp;- 紧张时... ``` | 子项只缩进 1 空格，或用单个 Tab |

⚠️ **Tab 注意**：1 个 Tab 算 1 个空白字符，需 2 个 Tab 才满足 `\s{2,}`。**建议统一用 2 空格缩进**（技能样本即如此）。

### 2.4 表格必须是标准 GFM 管道表

**要求**：
- 每行以 `|` 开头
- 第 2 行是分隔行，每个单元格 ≥1 个 `-`（可带 `:` 对齐符）

| 正确 | 错误 |
|------|------|
| `\| 章节 \| 标题 \|`<br>`\|------\|------\|`<br>`\| 第 1 章 \| 困兽 \|` | 缺分隔行 → 表头被当数据行 |
| `\| :--- \| ---: \|`（带对齐符也行） | 单元格用空格而非 `\|` 分隔 |

### 2.5 章号必须用阿拉伯数字

**正则**：`第\s*(\d+)\s*章`

| 正确 | 错误（章号解析为 null） |
|------|------------------------|
| `第 1 章` / `第1章` | `第一章`（中文数字） |

**影响文件**：细纲每章标题、核心情节每章、章节进度表行、大纲逐章节奏表行、rhythmData 的 chapter 字段。

⚠️ **卷号例外**：`第1卷` 和 `第一卷` **都认**（parseVolumeNumber 双兼容）。

### 2.6 版本头自动保留

文件首到第一个 `# ` 之间的内容（通常是 `**版本**：...` + `**修改记录**：...`）被视为 versionHeader，**原样保留，不参与解析**。改文件时不会被破坏。

## 三、节奏图谱.html 硬性规则

**文件**：`图解/节奏图谱.html`
**解析器**：[rhythm-html.ts](../src/main/data/skill-format/rhythm-html.ts)

### 3.1 rhythmData 数组块

必须存在形如 `const rhythmData = [ ... ];` 的块（正则 `/const\s+rhythmData\s*=\s*\[([\s\S]*?)\];/`）。html 其余部分（ECharts 配置、样式、表格）原样保留，不参与解析。

### 3.2 每条 entry 必须是精确的 JS 字面量

**正则**：
```
\{\s*chapter:\s*(\d+)\s*,\s*title:\s*'([^']*)'\s*,\s*emotion:\s*(\d+(?:\.\d+)?)\s*,\s*climax:\s*(\d+(?:\.\d+)?)\s*,\s*volume:\s*(\d+)\s*,\s*actualized:\s*(true|false)\s*\}
```

**正确**：
```javascript
{ chapter: 1, title: '困兽', emotion: 5, climax: 1, volume: 1, actualized: false }
```

**会导致该条被丢的写法**：
| 偏离 | 后果 |
|------|------|
| `title: "困兽"`（双引号） | 不匹配（正则要求单引号） |
| 字段顺序不同（如 emotion 在 title 前） | 不匹配 |
| `actualized: TRUE`（大写） | 不匹配（正则要求小写 `true`/`false`） |
| 缺少任一字段 | 不匹配 |
| title 内含未转义的单引号 `'` | 截断 |

**字段约束**：
- `chapter` / `volume`：整数
- `emotion` / `climax`：数字（整数或小数，如 `3.5`）
- `actualized`：`true` / `false`（小写）

## 四、逐文件详细要求

### 4.1 大纲/大纲.md

**解析器**：[project-skill-repo.ts](../src/main/data/skill-format/project-skill-repo.ts) + [outline-md-repo.ts](../src/main/data/skill-format/outline-md-repo.ts)

#### H1（书名）
```
# 《末日也要先领证》大纲
```
- 从 `《》` 中提取书名；无书名号时去掉末尾「大纲」二字。

#### `## 基本信息`（项目元信息）
```
## 基本信息
- **题材**：末日生存 + 先婚后爱
- **预计章节数**：300 章
- **每章字数**：约 2500 字
- **每章标准字数**：2500 字      ← 备用键名
- **简介**：...                  ← 备用：作品简介
```
- **题材** → genre
- **预计章节数**（取首个数字）→ targetChapters
- **每章字数** 或 **每章标准字数**（取首个数字）→ chapterWordCount
- **简介** 或 **作品简介** → description

#### `## 主线剧情走向`（卷结构 + 总纲）
```
## 主线剧情走向

（这段引言文字 → MainOutline.synopsis，可为空）

### 第一卷：天降老婆（第 1-30 章）
- **核心冲突**：...
...
### 第二卷：铁壁城（第 31-60 章）
```
- **synopsis** = 首个 H3 之前的引言段（无引言则为空）
- **volumes** = 每个 `### 第N卷：卷名（第X-Y章）`：
  - 卷号：parseVolumeNumber（`第一卷`/`第1卷` 都行）
  - 卷名：`：` 和 `（` 之间
  - 章节范围：`第X-Y章`（取 X、Y）

#### `## 逐章节奏标注`（逐章节奏回退表）
```
## 逐章节奏标注

### 第 1 卷（第 1-30 章）

| 章节 | 标题 | 情绪值 | 爽点类型 | 卷 |
|------|------|--------|----------|------|
| 第 1 章 | 困兽 | 5 | 1 | 1 |
```
- 每卷一个 H3 子节，内含一张表
- **列头必须含**：`章节` / `标题` / `情绪` / `爽点` / `卷`（子串匹配）
- 此表是 rhythmData html 缺失时的**回退源**；有 html 时以 html 为准

### 4.2 细纲/第NN卷.md

**解析器**：[detailed-outline-md-repo.ts](../src/main/data/skill-format/detailed-outline-md-repo.ts)
**文件名约定**：`第NN卷.md`（NN 零填充 2 位，如 `第01卷.md`）

#### H1（卷号来源）
```
# 第 1 卷 细纲：天降老婆（第 1-30 章）
```
- 用 parseVolumeNumber 提取卷号（**此处样本用阿拉伯**）

#### 每章 H2
```
## 第 1 章：困兽
- **节奏标注**：
  - 情绪值：5
  - 爽点类型：1（小打脸）
- **核心事件**：孢子灾害初现...
- **爽点/打脸**：苏铭用...砸退...     ← 或 **爽点**
- **角色出场**：
  - 苏铭（首次正式登场）
  - 图书馆管理员老周（路人，戴歪棒球帽）
- **伏笔铺设**：
  - 窗外天突然变绿
- **章末钩子**：窗外的变异兽群...
- **字数预估**：约 2500 字
- **金句**：「听我的。」
- **卷终反转**：...                    ← 或 **关键设定**（→ climaxTag）
```
- 章号：parseChapterNumber（**必须阿拉伯** `第 1 章`）
- 标题：`：` 之后、`（` 之前
- `节奏标注` 子列表内用 `情绪值：N` / `爽点类型：N` 提取数字
- 字段名映射：`核心事件`→plotSummary、`爽点/打脸`|`爽点`→coolPoint、`角色出场`→charactersAppearing、`伏笔铺设`→foreshadowings、`章末钩子`→hook、`字数预估`→wordEstimate、`金句`→goldenLine、`卷终反转`|`关键设定`→climaxTag

### 4.3 记忆系统/角色卡.md

**解析器**：[character-card-md-repo.ts](../src/main/data/skill-format/character-card-md-repo.ts)

#### 分类 H2（必须是这三个确切名）
```
## 主角
## 核心配角
## 核心反派
```
⚠️ **`## 路人角色记录` 不解析**（结构不同，Phase 1 起跳过）。

#### 角色 H3
```
### 苏铭（男主）
### 老吴                            ← 无括号也行
### 沈惊蛰（前期反派）
```
- 名字 = `### ` 之后、`（` 或行尾
- 子类型 = `（...）` 内（中/英括号都认）
- **role** = `${分类}·${子类型}`，如 `主角·男主`

#### 字段块（parseBoldFields）
任意 `- **字段**：值` 字段都会进 rawFields。常用：姓名 / 身份 / 显性性格 / 隐性性格 / 行为习惯（多行）/ 金手指 / 觉醒路线 / 成长目标（多行）/ 专属标签 / 当前状态 / 核心动机 / 登场章节。

- **专属标签**：按 `#tag#` 模式提取（如 `#末日也要先领证# #闷骚护妻狂魔#` → tags 数组）
- 其余字段全部保留在 `rawFields`，UI 用 MarkdownView 完整展示

#### `## 关系变更日志`（可选，关系数据源）
```
## 关系变更日志

| 章节 | 角色A | 角色B | 变更前 | 变更后 | 触发事件 |
|------|-------|-------|--------|--------|----------|
| 8 | 苏铭 | 老吴 | 陌生 | 亦师亦友 | 街角偶遇 |
```
- **列头必须含**：`章节` / `角色A` / `角色B` / `变更前` / `变更后` / `触发`
- 每行 → 一条 Relationship（relationType=变更后，description=变更轨迹+触发事件）
- characterAId/BId 用与角色一致的 charId 哈希

### 4.4 记忆系统/地点档案.md

**解析器**：[location-md-repo.ts](../src/main/data/skill-format/location-md-repo.ts)

#### 每个地点 H2

```
## 1. 大学图书馆
## 2. 城南安全屋（徐昭昭未来准备）
```
- 序号前缀 `\d+\s*[.、)]?` 自动剥离得 name

#### 字段块
```
- **类型**：城市建筑
- **特征描述**：5 层开放式阅览空间...

```

### 4.6 记忆系统/伏笔追踪.md

**解析器**：[foreshadowing-md-repo.ts](../src/main/data/skill-format/foreshadowing-md-repo.ts)

⚠️ **关键**：伏笔大表在**文档顶层**（H1 下、无 H2 包裹），解析器从 `doc.body` 取第一个表。

#### 表列头（子串匹配）
```
| 伏笔编号 | 伏笔内容 | 伏笔类型 | 埋设章节 | 预计回收章节 | 实际回收章节 | 状态 |
|----------|----------|----------|----------|-------------|-------------|------|
| FB-001 | 未来婚戒 | 道具 | 第 3 章 | 第 282 章 | 未回收 | 未回收 |
```
- **编号**（列头含「编号」）→ id，必须形如 `FB-NNN`（create 时自动生成）
- **内容**（含「内容」）→ content
- **类型**（含「类型」）→ note
- **埋设**（含「埋设」）→ plantChapter（取首个数字）
- **预计**+**回收** → expectedCollect
- **实际**+**回收** → actualCollect
- **状态** → status（见下）

#### 状态文本映射（mapStatus）
| v3.2 文本（子串匹配） | app 枚举 |
|----------------------|---------|
| 含「已回收」 | collected |
| 含「已错过」或「遗漏」 | missed |
| 含「已埋设」「部分回收」「强化」 | planted |
| 其他（含「未回收」） | pending |

#### 写回（unmapStatus）
`collected`→`已回收` / `missed`→`已错过` / `planted`→`已埋设` / `pending`→`未回收`

> 关键伏笔详解（`### FB-XXX ...`）不参与结构化解析，原样保留。

## 五、列头关键词依赖汇总

所有表格解析用 `findIndex(h => h.includes(keyword))` 子串匹配。列名必须含这些关键词：

| 表（文件） | 必须含的关键词 |
|-----------|--------------|
| 大纲逐章节奏标注 | `章节` `标题` `情绪` `爽点` `卷` |
| 角色卡·关系变更日志 | `章节` `角色A` `角色B` `变更前` `变更后` `触发` |
| 伏笔追踪 | `编号` `内容` `类型` `埋设` `预计`+`回收` `实际`+`回收` `状态` |
| 章节进度 | `章节` `标题` `状态` `字数` `备注` |

⚠️ **这是最脆的耦合点**。技能改列名（如「伏笔内容」→「内容描述」）→ 该列数据丢。建议技能侧锁死列名。

## 六、容错行为（优雅降级，不崩）

| 偏离 | 行为 |
|------|------|
| 文件不存在 | 返回 `[]` / `null` / 空 Map |
| 某字段缺失 | 该字段 `undefined`，其他字段正常 |
| 表格多/少列 | 按表头列数补齐/截断 |
| 表头名变了（不含关键词） | 该列返回 -1，对应字段 undefined |
| 未知 H2 节 | 读：忽略；写：原样保留 |
| 中文/英文冒号、中/英括号 | 都认 |
| 卷号中文/阿拉伯 | 都认 |
| 代码围栏里的 `##` | 不误切 |

## 七、风险清单（按严重度）

### 🔴 高
1. **列名改动**（第五节）—— 整列数据丢，静默。建议：解析后做非空校验，0 条伏笔/角色时 UI 给黄条提示。
2. **rhythmData entry 格式偏离**（第三节）—— 单条丢。任何字段顺序/引号/大小文变化都失效。
3. **字段行去掉 `- **`**（第 2.2 节）—— 角色卡/地点档案字段全丢。

### 🟡 中
4. **子列表用 Tab 缩进**（第 2.3 节）—— 多行字段（行为习惯/成长目标/关联事件）丢。需 2 空格。
5. **章号用中文数字**（第 2.5 节）—— 细纲/核心情节/进度表该章丢。卷号不受影响。
6. **标题缺空格** `##标题`（第 2.1 节）—— 整节丢。

### 🟢 低
7. **版本头格式** —— 不影响解析（原样保留），但 Phase 3 写回不自动 bump 版本号。
8. **问题记录/提示词格式** —— 当前不解析，任意格式都行。
